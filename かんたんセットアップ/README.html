かんたんセットアップ（ダブルクリックだけで環境構築）

このフォルダのファイルを使うと、社内つぶやきボードの 開発環境 をあなたの PC で準備できます。

操作は基本 ファイルをダブルクリックするだけ です。 内部では WSL や Podman という裏方の仕組みを使っていますが、その中身を理解する必要はありません。この研修の目的は Codex CLI を使った開発を体験する ことです。下の手順どおりに進めてください。

💡 困ったときは、Oracle削除.bat を実行してから Oracle起動.bat をやり直すと、たいていの問題はリセットできます。

1. 準備するもの

  • Windows 10 または 11（64bit）のパソコン
  • そのパソコンの 管理者 として使えるアカウント
  • インターネット接続
  • ディスクの空き容量 10 GB 以上
  • 講師から渡された OPENAI_API_KEY（sk- で始まる文字列）

2. 最初の 1 回だけ行う手順

順番にダブルクリックしていくだけです。

順番 ファイル やること
① セットアップ1_Windows準備.bat Windows 側の準備（要 管理者）。終わったら PC を再起動
②（手動） スタートメニュー →「Ubuntu」 初回起動。ユーザー名とパスワードを自分で決めて入力
③ セットアップ2_Ubuntu準備.bat 開発ツール（JDK / Maven / Podman / Codex 環境）の導入
④ セットアップ3_APIキー設定.bat OPENAI_API_KEY と設定ファイルを登録
⑤ 環境チェック.bat 準備ができているか診断（PASS が並べば OK）
⑥ Oracle起動.bat データベースを起動（初回は数分）

⚠️ パスワードや API キーは 入力中に画面へ表示されません（* も出ません）。でも、ちゃんと入力されています。

3. 毎日の使い方

ファイル やること
Oracle起動.bat その日の作業を始めるとき。データベースを起動します
Oracle停止.bat 作業を終えるとき。データは残ります

4. 接続情報

Oracle起動.bat の成功時に画面へ表示されます。

  • URL: jdbc:oracle:thin:@//localhost:1521/XEPDB1
  • ユーザー名: tsubuyaki
  • パスワード: .env の ORACLE_APP_PWD（デフォルト tsubuyaki_pw）

5. Codex CLI を使う（研修の本題）

環境構築はバッチに任せていますが、Codex CLI の起動はあえてバッチにしていません。 Codex を使った開発こそがこの研修の目的なので、Ubuntu のターミナルで自分でコマンドを打って起動します。

codex-shell        # Codex 開発コンテナに入る
codex --help       # まずはヘルプを表示してみる

6. うまくいかないとき

  • Ubuntu がスタートメニューに見つからない → 手順①のあと、PC の再起動がまだかもしれません。再起動してからもう一度探してください。
  • パスワードがはじかれる → 入力中は画面に出ないので、ゆっくり正確に入力してください。
  • Oracle起動.bat が成功しない → Oracle削除.bat → Oracle起動.bat の順でリセットしてください。
  • それでも直らないときは 環境チェック.bat を実行し、表示された案内に従ってください。

7. 研修が終わったら（あと片付け）

研修の最終日に PC をきれいに戻したいときだけ使います。

ファイル やること
研修終了_環境削除.bat 開発環境（WSL の Ubuntu-22.04）を丸ごと削除します。Oracle のデータ・書いたコード・開発ツール・API キーがすべて一緒に消えます

⚠️ 元に戻せません。 実行すると確認のため半角で delete と入力を求められます。Windows 側のアプリや C:\workspace などのフォルダは残ります。もう一度使うには手順①からやり直してください。
