# セルフレビュー (XSS / SQLi / ハードコード)

PR を出す前の Codex 自己レビュー用プロンプト。3 観点で diff をスキャンし、
NG があればその場で修正案を出させる。

---

タスク: 現在の git diff (まだコミットしていない変更を含む) を自己レビューしてください。

観点:
1. **XSS**: Thymeleaf テンプレートで `th:utext` を使っていないか。動的文字列を `<script>` 内に出力していないか。`@ResponseBody` で HTML を返していないか。
2. **SQL インジェクション**: Repository / Service で文字列連結された JPQL/SQL を使っていないか。`@Query` の bind 変数を使っているか。動的検索条件で `String.format` していないか。
3. **ハードコード**: パスワード、API キー、接続情報、本番エンドポイント URL がソースに直書きされていないか。`@Value` または環境変数経由になっているか。

報告フォーマット:
```
### XSS
- OK / NG (NG なら以下を続ける)
- ファイル: <path>:<line>
- 現在のコード: <抜粋>
- 修正案: <抜粋>

### SQL インジェクション
- OK / NG (同上)

### ハードコード
- OK / NG (同上)

### 結論
- コミットしてよい / NG を直してから再レビュー
```

3 観点すべて OK でない限り、コミットを進めないでください。
